一、 场景设定与高层视角剖析
汇报对象： 沙特城乡事务与住房部（MOMRAH）部长，或利雅得市长。
当前时间背景： 2026年3月9日（Dashboard左上角时间）。此时正是利雅得人口突破810万[1][2]、萨勒曼国王公园（King Salman Park）一期即将向公众开放的关键节点[3]。
领导核心关切（痛点）：
政治任务： “绿色利雅得（Green Riyadh）”倡议要求人均绿地面积从1.7㎡提升至28㎡[4][5]。巨资投入的萨勒曼国王公园（16.6平方公里，世界最大城市公园[3][6]）能否真正辐射全城？
投资回报： 公园周边的土地价值（Zoning）如何重塑？基础设施缺口在哪里？
市民感知： 巨型项目如何转化为普通市民“15分钟生活圈”内的真实获得感？

三、 极简且致命的故事线设计（Golden Thread）
整个故事分为三段式：“宏观确信” -> “微观痛点揭示” -> “AI驱动的精准重塑”。
1. 序章：从 Dashboard 开启全局视角（The Hook）
操作： 站在大屏前，引导领导目光看向左上角和正中央。
演讲脚本：
“尊敬的阁下，早上好。今天是2026年3月9日。看着大屏左上方，利雅得的人口已正式突破810万[1][2]。随着城市的极速扩张，如何保持城市的‘健康度’（指向左侧 Health Index 85）是我们最大的挑战。
请看屏幕中央，我们的数字孪生系统正实时监控着‘2030愿景’的核心地标——萨勒曼国王公园。系统评估其与国家愿景的契合度高达98.5%。它不仅是16.6平方公里的世界最大城市公园[3][6]，更是利雅得的‘绿色心脏’。
但是，阁下，一个尖锐的问题是：这颗强大的心脏，它的绿色血液能否输送到城市的每一根毛细血管？市民真的能轻易到达那里吗？ 让我们进入系统第二幕‘空间优化’，看看 AI 给出的残酷真相和解决方案。”
操作： 点击 Dashboard 底部的 OPTIMIZATION（优化）Tab，或者直接点击 3D 图上的 Optimization Focus 标签，进入内页。
2. 深入内页：揭示痛点与机遇（The Problem）
内页设计构想（依据PDF第8、9页：页面6&7规范整合）：
左侧：【15分钟等时圈覆盖地图（Isochrone Map）】。以萨勒曼国王公园为中心，画出步行/骑行/公交15分钟能到达的区域。灰色代表未覆盖盲区。
右上方：【绿地缺口热力图与数据】。
右下方：【AI 优化建议（Agent Recommendations）】。
演讲脚本：
“阁下，这是利雅得核心区的15分钟可达性等时圈。绿色的部分是目前能够通过步行和微公交在15分钟内享受到公园红利的区域。如您所见（指着地图上的大片灰色区域），由于周边快速路（如 Abu Bakr Al-Siddiq 路[9]）的割裂，公园西侧和南侧的高密度住宅区，实际上成为了**‘绿色孤岛’**。
我们的智能体（Parks Selector Agent）计算出，尽管我们建成了世界最大的公园，但该区域500米服务半径覆盖的人口比例仅为52%（引用PDF数据）。‘绿色利雅得’要求人均绿地达到28平米[4][5]，如果我们不进行空间微调，巨额的基建投资将无法转化为这近半数市民的满意度。”
3. 破局：智能体的精准干预与 ROI 展示（The Solution & ROI）
操作： 在地图上点击智能体生成的“优化预测（Optimization Simulation）”按钮。
内页动态展示： 地图上浮现出几个发光的节点（微型公园/口袋公园选址），同时等时圈（绿色区域）迅速扩大，覆盖了原本的灰色区域。右侧的柱状图显示土地价值预期飙升。
演讲脚本：
“但我们不需要盲目拆建。这是系统中的 Zoning Mix Advisor Agent 联合 Accessibility Optimizer 给出的最优解。
系统在公园外围2公里内的闲置地块（Idle Lands）中，精准锚定了 8个高潜力口袋公园和3条慢行绿道连廊的选址（引用PDF第9页数据）。
阁下，请看右侧的效益评估雷达：
覆盖率跃升： 仅需投入极少的改造成本，15分钟生活圈覆盖人口将从52%瞬间跃升至78%。新增服务人口超15万人。
土地价值释放： 连廊打通后，周边的商业与居住用地价值预计提升15%-20%（引用PDF第12页数据）。
碳减排与温度控制： 连片的绿网将使该区域的微气候温度再降低1.5度[5]。
我们不仅在建一个公园，我们是在用 AI 规划网络的手段，花最低的成本，将萨勒曼国王公园的红利‘杠杆化’，送到每一位利雅得市民的家门口。”
四、 内页 (Optimization Detail Page) 的 UI 落地设计指导
为了完美支撑上述故事，当从主 Dashboard 切换到内页时，UI 应呈现出“高管驾驶舱”的质感，绝不能是密密麻麻的Excel表。
布局规范（遵循PDF设计建议）：
主视觉（占据左侧 60% 面板）：多图层 GIS 叠加地图。
底图：暗黑风格的利雅得 GIS 地图。
图层 1：萨勒曼国王公园的深绿色高亮实体。
图层 2：发光的 15-min 等时圈（当前状态为淡蓝色，优化后叠加一层扩张的霓虹绿色轮廓）。
图层 3：8个脉冲发光的点（AI 建议的微型连廊/口袋公园选址）。
右侧顶部（20% 面板）：核心指标对比（Before vs After）
使用大号数字字体，例如：
15-min Population Covered: 52% -> <span style="color: green">78%</span>
Green Space per Capita: 1.7 m² -> <span style="color: green">12.5 m² (该区域)</span>
右侧中部（20% 面板）：土地价值与 ROI 柱状图 / 雷达图
X轴为区域，Y轴为预测的价值提升百分比（Res: +15%, Com: +20%）。
底部（全宽的狭长面板）：Agent 行动执行栏
显示 Amanahs (智能规划引擎) Status: Ready。
一个具有仪式感的按钮：[ GENERATE ZONING RESTRUCTURE REPORT ] (生成区划重组报告) 或 [ EXPORT TO MUNICIPALITY ]。
五、 给演示者的实战 Tips
眼球控制： 领导通常只看屏幕前15秒。一定要从 Dashboard 上那个大大的、发光的 KING SALMAN PARK (98.5% Active) 开始讲，这是他们的政绩骄傲。先肯定成绩，再引出挑战。
术语使用： 在沙特做汇报，多使用这些缩写和词汇会极大增加好感和专业度：Vision 2030, Quality of Life (QoL) Program, Green Riyadh, Salmani Architecture（萨勒曼建筑风格）, ROI, Micro-climate, Data-driven Zoning。
对比的力量： 领导不在乎你的算法有多复杂，他们只关心“做了这个动作前”和“做了之后”的指标对比。手指要在屏幕上“灰色盲区”和AI生成的“绿色连廊”之间划过，形成强烈的视觉与逻辑反差。
真实感约束： 绝不提“我们随便造了几个数据”。每次提到人口、面积、公园名称，语气都要笃定，因为这些都是基于沙特2026年现状的绝对真实投影。
